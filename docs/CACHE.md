# Caching and Memoization

The caching feature allows you to cache action results based on input to avoid redundant processing. This is particularly useful for expensive operations like database queries, external API calls, or complex computations.

## Basic Usage

Add caching to any action by calling the `.cache()` method:

```typescript
import { action } from '@arqetype/next-validated-action';
import { IsString } from 'class-validator';

class UserInput {
  @IsString()
  userId!: string;
}

export const getUserProfile = action
  .inputDto(UserInput)
  .cache({
    ttl: 60000, // Cache for 60 seconds
  })
  .action(async ({ parsedInput }) => {
    // This expensive database query will be cached
    const user = await db.user.findUnique({
      where: { id: parsedInput.userId },
    });
    return user;
  });
```

## Configuration Options

The `cache()` method accepts a configuration object with the following options:

```typescript
interface CacheConfig<TInput> {
  /**
   * Time to live in milliseconds
   * If not specified, cached values never expire
   */
  ttl?: number;

  /**
   * Custom cache key generator function
   * Default: Uses JSON.stringify of the input
   */
  key?: (input: TInput) => string;

  /**
   * Storage backend for cache
   * Default: 'memory' (uses global in-memory storage)
   */
  storage?: CacheStorage | 'memory';

  /**
   * Whether to cache error results
   * Default: false (only successful results are cached)
   */
  cacheErrors?: boolean;
}
```

## Cache Key Generation

### Default Key Generation

By default, cache keys are generated by hashing the JSON representation of the input:

```typescript
const getUser = action
  .inputDto(UserInput)
  .cache({ ttl: 60000 })
  .action(async ({ parsedInput }) => {
    // Cache key is automatically generated from input
    return await fetchUser(parsedInput.userId);
  });
```

### Custom Key Generation

For more control over cache keys, provide a custom key generator:

```typescript
class PostInput {
  @IsString()
  postId!: string;

  @IsNumber()
  version!: number;
}

const getPost = action
  .inputDto(PostInput)
  .cache({
    ttl: 300000, // 5 minutes
    // Only use postId for cache key, ignore version
    key: (input) => `post:${input.postId}`,
  })
  .action(async ({ parsedInput }) => {
    return await db.post.findUnique({
      where: { id: parsedInput.postId },
    });
  });
```

### Benefits of Custom Keys

- **Ignore certain fields**: Don't invalidate cache for non-essential input changes
- **Namespace keys**: Organize cache keys with prefixes
- **Include context**: Add user IDs or tenant IDs to keys
- **Optimize storage**: Create shorter, more efficient keys

## Storage Backends

### In-Memory Storage (Default)

The default storage uses a global in-memory cache with automatic cleanup:

```typescript
const cachedAction = action
  .cache({
    ttl: 60000,
    storage: 'memory', // This is the default
  })
  .action(async ({ parsedInput }) => {
    // ...
  });
```

**Characteristics:**

- Fast access
- Automatically cleans up expired entries
- Shared across all actions in the same process
- Lost on server restart
- Not suitable for serverless environments with multiple instances

### Custom Storage

Create your own storage implementation:

```typescript
import { MemoryCacheStorage } from '@arqetype/next-validated-action';

// Create a dedicated cache instance for this action
const myCache = new MemoryCacheStorage();

const cachedAction = action
  .cache({
    ttl: 60000,
    storage: myCache,
  })
  .action(async ({ parsedInput }) => {
    // ...
  });

// Later, you can clear this specific cache
await myCache.clear();

// Get cache statistics
const stats = myCache.getStats();
console.log(`Cache size: ${stats.size}, expired: ${stats.expired}`);

// Don't forget to stop cleanup when done
myCache.stopCleanup();
```

## Caching Errors

By default, only successful results are cached. Failed actions will be re-executed on the next call.

### Cache Successful Results Only (Default)

```typescript
const unreliableAPI = action
  .cache({
    ttl: 60000,
    cacheErrors: false, // Default
  })
  .action(async ({ parsedInput }) => {
    const result = await fetch('https://api.example.com/data');
    if (!result.ok) {
      throw new Error('API failed');
    }
    return result.json();
  });

// First call fails - not cached
await unreliableAPI(input); // Error

// Second call - tries again
await unreliableAPI(input); // Might succeed
```

### Cache All Results Including Errors

```typescript
const expectedToFail = action
  .cache({
    ttl: 60000,
    cacheErrors: true, // Cache errors too
  })
  .action(async ({ parsedInput }) => {
    throw new Error('This always fails');
  });

// First call fails - cached
await expectedToFail(input); // Error

// Second call - returns cached error immediately
await expectedToFail(input); // Same error, no re-execution
```

**Note:** Validation errors (both input and output) are cached when `cacheErrors: true` is set.

## Cache Expiration (TTL)

### With TTL

Values expire after the specified time:

```typescript
const shortLivedCache = action
  .cache({
    ttl: 5000, // 5 seconds
  })
  .action(async ({ parsedInput }) => {
    return await expensiveOperation(parsedInput);
  });

await shortLivedCache(input); // Executes action
await shortLivedCache(input); // Uses cache (within 5s)

// Wait 6 seconds
await new Promise((resolve) => setTimeout(resolve, 6000));

await shortLivedCache(input); // Cache expired, executes again
```

### Without TTL

Values never expire (until cache is cleared or server restarts):

```typescript
const permanentCache = action
  .cache({
    // No TTL - cached forever
  })
  .action(async ({ parsedInput }) => {
    return await fetchStaticData(parsedInput);
  });
```

## Custom Storage Implementation

Implement the `CacheStorage` interface for custom backends:

```typescript
import { CacheStorage } from '@arqetype/next-validated-action';

class RedisCacheStorage implements CacheStorage {
  constructor(private redis: RedisClient) {}

  async get<T>(key: string): Promise<T | undefined> {
    const value = await this.redis.get(key);
    return value ? JSON.parse(value) : undefined;
  }

  async set<T>(key: string, value: T, ttl?: number): Promise<void> {
    const serialized = JSON.stringify(value);
    if (ttl) {
      await this.redis.setex(key, Math.ceil(ttl / 1000), serialized);
    } else {
      await this.redis.set(key, serialized);
    }
  }

  async delete(key: string): Promise<void> {
    await this.redis.del(key);
  }

  async clear(): Promise<void> {
    await this.redis.flushdb();
  }

  async has(key: string): Promise<boolean> {
    const exists = await this.redis.exists(key);
    return exists === 1;
  }
}

// Use with actions
const redisCache = new RedisCacheStorage(redisClient);

const cachedAction = action
  .cache({
    ttl: 60000,
    storage: redisCache,
  })
  .action(async ({ parsedInput }) => {
    // ...
  });
```

## Best Practices

### 1. Set Appropriate TTL

```typescript
// Short TTL for frequently changing data
const recentPosts = action.cache({ ttl: 10000 }); // 10 seconds

// Medium TTL for moderate change frequency
const userProfile = action.cache({ ttl: 300000 }); // 5 minutes

// Long TTL for rarely changing data
const siteConfig = action.cache({ ttl: 3600000 }); // 1 hour
```

### 2. Use Meaningful Cache Keys

```typescript
const getUserData = action
  .cache({
    ttl: 60000,
    // Include relevant context in the key
    key: (input) => `user:${input.userId}:profile`,
  })
  .action(async ({ parsedInput }) => {
    // ...
  });
```

### 3. Consider Cache Size

```typescript
// Create separate caches for different purposes
const userCache = new MemoryCacheStorage();
const postCache = new MemoryCacheStorage();

const getUser = action.cache({ storage: userCache });
const getPost = action.cache({ storage: postCache });

// Monitor cache size
setInterval(() => {
  const userStats = userCache.getStats();
  if (userStats.size > 1000) {
    console.warn('User cache is getting large:', userStats);
  }
}, 60000);
```

### 4. Cache Expensive Operations Only

```typescript
// Good: Cache expensive database queries
const complexQuery = action
  .cache({ ttl: 60000 })
  .action(async ({ parsedInput }) => {
    return await db.query.complex(parsedInput);
  });

// Bad: Don't cache simple operations
const simpleCalc = action
  .cache({ ttl: 60000 }) // Unnecessary overhead
  .action(async ({ parsedInput }) => {
    return parsedInput.a + parsedInput.b; // Too simple to cache
  });
```

### 5. Handle Cache Invalidation

```typescript
// Clear cache when data changes
const updateUser = action.action(async ({ parsedInput }) => {
  const result = await db.user.update(parsedInput);

  // Invalidate user cache
  const userCache = getGlobalMemoryCache();
  await userCache.delete(`user:${parsedInput.id}:profile`);

  return result;
});
```

## Examples

### Example 1: Caching External API Calls

```typescript
import { action } from '@arqetype/next-validated-action';
import { IsString } from 'class-validator';

class WeatherInput {
  @IsString()
  city!: string;
}

export const getWeather = action
  .inputDto(WeatherInput)
  .cache({
    ttl: 600000, // Cache for 10 minutes
    key: (input) => `weather:${input.city.toLowerCase()}`,
  })
  .action(async ({ parsedInput }) => {
    const response = await fetch(
      `https://api.weather.com/v1/current?city=${parsedInput.city}`
    );
    return response.json();
  });
```

### Example 2: User-Specific Caching

```typescript
class UserDashboardInput {
  @IsString()
  userId!: string;
}

export const getUserDashboard = action
  .inputDto(UserDashboardInput)
  .needsAuth(async () => getCurrentUser())
  .cache({
    ttl: 30000, // 30 seconds
    key: (input) => `dashboard:user:${input.userId}`,
  })
  .action(async ({ parsedInput }) => {
    // Expensive aggregation query
    const [stats, recentActivity, notifications] = await Promise.all([
      db.stats.getUserStats(parsedInput.userId),
      db.activity.getRecent(parsedInput.userId, 10),
      db.notifications.getUnread(parsedInput.userId),
    ]);

    return { stats, recentActivity, notifications };
  });
```

### Example 3: Caching with Retry

```typescript
export const resilientAPICall = action
  .inputDto(APIInput)
  .cache({
    ttl: 120000, // Cache for 2 minutes
    cacheErrors: false, // Retry on errors
  })
  .retry({
    attempts: 3,
    delay: 1000,
    backoff: 'exponential',
  })
  .action(async ({ parsedInput }) => {
    const response = await fetch(parsedInput.url);
    if (!response.ok) {
      throw new Error(`API error: ${response.status}`);
    }
    return response.json();
  });
```

### Example 4: Multi-Tenant Caching

```typescript
class TenantDataInput {
  @IsString()
  tenantId!: string;

  @IsString()
  dataType!: string;
}

export const getTenantData = action
  .inputDto(TenantDataInput)
  .cache({
    ttl: 300000, // 5 minutes
    key: (input) => `tenant:${input.tenantId}:${input.dataType}`,
  })
  .action(async ({ parsedInput }) => {
    return await db.tenantData.find({
      where: {
        tenantId: parsedInput.tenantId,
        type: parsedInput.dataType,
      },
    });
  });
```

## API Reference

### `action.cache(config)`

Configures caching for the action.

**Parameters:**

- `config: CacheConfig<TInput>` - Cache configuration object

**Returns:** New `ActionClientBuilder` instance with caching enabled

### `CacheConfig<TInput>`

```typescript
interface CacheConfig<TInput> {
  ttl?: number;
  key?: (input: TInput) => string;
  storage?: CacheStorage | 'memory';
  cacheErrors?: boolean;
}
```

### `CacheStorage` Interface

```typescript
interface CacheStorage {
  get<T>(key: string): Promise<T | undefined>;
  set<T>(key: string, value: T, ttl?: number): Promise<void>;
  delete(key: string): Promise<void>;
  clear(): Promise<void>;
  has(key: string): Promise<boolean>;
}
```

### `MemoryCacheStorage`

In-memory cache implementation with automatic cleanup.

**Constructor:**

```typescript
new MemoryCacheStorage(cleanupIntervalMs?: number)
```

**Methods:**

- `get<T>(key: string): Promise<T | undefined>` - Get cached value
- `set<T>(key: string, value: T, ttl?: number): Promise<void>` - Set cached value
- `delete(key: string): Promise<void>` - Delete cached value
- `clear(): Promise<void>` - Clear all cached values
- `has(key: string): Promise<boolean>` - Check if key exists
- `size(): number` - Get number of cached items
- `getStats(): { size: number; expired: number }` - Get cache statistics
- `stopCleanup(): void` - Stop automatic cleanup (call when done)

### Utility Functions

#### `getGlobalMemoryCache()`

Returns the global memory cache instance (creates one if it doesn't exist).

```typescript
import { getGlobalMemoryCache } from '@arqetype/next-validated-action';

const cache = getGlobalMemoryCache();
await cache.clear(); // Clear all cached actions
```

#### `clearGlobalMemoryCache()`

Clears and stops the global memory cache.

```typescript
import { clearGlobalMemoryCache } from '@arqetype/next-validated-action';

clearGlobalMemoryCache();
```

#### `generateDefaultCacheKey(input)`

Generates a cache key from input (same algorithm used by default).

```typescript
import { generateDefaultCacheKey } from '@arqetype/next-validated-action';

const key = generateDefaultCacheKey({ userId: '123', name: 'John' });
console.log(key); // "cache:a1b2c3d4"
```

## Performance Considerations

### Memory Usage

The in-memory cache stores all cached values in RAM. Monitor memory usage for applications with:

- High cache volume
- Large cached values
- Long TTLs

### Cleanup Overhead

The `MemoryCacheStorage` runs automatic cleanup every 60 seconds by default. For high-frequency applications, consider:

- Adjusting cleanup interval
- Using shorter TTLs
- Implementing custom storage with more efficient expiration

### Concurrent Requests

The current implementation does not deduplicate concurrent requests with the same cache key. If this is needed, consider implementing request deduplication middleware.

## Troubleshooting

### Cache Not Working

1. **Verify cache is configured:**

   ```typescript
   const myAction = action
     .cache({ ttl: 60000 }) // Must call .cache()
     .action(async () => {
       /* ... */
     });
   ```

2. **Check cache key uniqueness:**

   ```typescript
   .cache({
     key: (input) => {
       console.log('Cache key:', `user:${input.id}`);
       return `user:${input.id}`;
     }
   })
   ```

3. **Verify TTL hasn't expired:**
   ```typescript
   .cache({
     ttl: 60000, // Make sure this is long enough
   })
   ```

### High Memory Usage

1. **Reduce TTL:**

   ```typescript
   .cache({ ttl: 30000 }) // Shorter TTL = less memory
   ```

2. **Use custom cache per action:**

   ```typescript
   const cache = new MemoryCacheStorage();
   // Monitor and clear when needed
   if (cache.size() > 1000) {
     await cache.clear();
   }
   ```

3. **Implement custom storage with size limits**

### Cache Stale Data

1. **Invalidate on updates:**

   ```typescript
   const cache = getGlobalMemoryCache();
   await cache.delete(cacheKey);
   ```

2. **Use shorter TTL:**

   ```typescript
   .cache({ ttl: 5000 }) // 5 seconds for frequently changing data
   ```

3. **Don't cache rapidly changing data**
